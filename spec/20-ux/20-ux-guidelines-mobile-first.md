# /spec/20-ux/20-ux-guidelines-mobile-first.md

## Guidelines mobile-first
- Largeur cible: 360px minimum
- Hit targets: 44px min
- Inputs scores: gros, lisibles, usage clavier numérique
- Navigation: bottom tabs (3 entrées)
- Feedback: toast + état visuel sur validation/correction
- États: loading, vide, erreur toujours prévus

## Prévention d’erreurs
- Un joueur ne peut être sélectionné qu’une fois par tour → UI doit bloquer (grisé + label)
- Boutons de validation désactivés tant que table invalide
